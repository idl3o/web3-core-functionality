Welcome to the Web3 Crypto Streaming Documentation

The Web3 Crypto Streaming Platform is a decentralized streaming service that empowers creators and provides viewers with transparent, blockchain-based content access. This documentation will guide you through integrating with our platform, whether you're a creator, developer, or platform partner.

Platform Overview

The Web3 Crypto Streaming Platform combines the best of blockchain technology with high-quality streaming capabilities. Key components include:

  • Decentralized Access Control - Content access rights stored on the blockchain for transparency and portability
  • Token-Based Ecosystem - STREAM tokens power the platform economy, enabling micropayments, creator rewards, and governance
  • IPFS Storage - Content is distributed across the InterPlanetary File System for censorship resistance
  • Web3 Authentication - Connect with crypto wallets for seamless identity verification
  • Creator Dashboard - Analytics and monetization tools for content creators

Getting Started

Follow these steps to begin working with the Web3 Crypto Streaming Platform:

  1. Set up a Web3 wallet - Create a wallet compatible with Ethereum/Polygon
  2. Get API credentials - Register on our developer portal to obtain API keys
  3. Install SDK - Choose the SDK for your preferred programming language
  4. Connect to the network - Configure your application to connect to our API endpoints
  5. Implement authentication - Enable wallet-based sign-in for your users

SDK Integrations

We provide a range of SDKs to make integration with our platform straightforward in your preferred programming language:

JavaScript/TypeScript SDK

Our JavaScript SDK provides comprehensive tools for browser and Node.js applications, with TypeScript type definitions included. 

npm install @web3-streaming/client

Python SDK

The Python SDK is ideal for backend services, data analysis, and developing content management tools. 

pip install web3-streaming-client

.NET SDK

For .NET developers, our C# SDK provides a strongly-typed interface to all platform functionality, compatible with .NET 6 and later.

API Access

Our RESTful API provides complete access to the platform's capabilities. Here's an example of a basic API request to check content access rights:

Check Content Access

Verifies if a user has valid access rights to a specific content item.

POST /api/v1/content/:contentId/access

Parameters

Name Type Required Description
contentId string Yes Unique identifier for the content
userAddress string Yes Ethereum address of the user

Response

{
  "hasAccess": true,
  "expiryTimestamp": 1713542400,
  "remainingTime": 3600,
  "tokenId": "stream_token_12345"
}

Smart Contracts

The Web3 Crypto Streaming Platform is built on a suite of smart contracts that handle token economics, access control, and payment distribution:

  • StreamingToken (ERC-20) - The platform's utility token that powers the economic system
  • StreamAccess - Manages content access rights on the blockchain
  • StreamPayment - Handles payment distribution between creators, platform, and other stakeholders

Here's a sample of the StreamAccess contract showing how access rights are verified: 

// SPDX-License-Identifier: MIT
pragma solidity ^0.8.9;

/**
 * @title StreamAccess
 * @dev Manages content access rights for the Web3 Streaming Platform
 */
contract StreamAccess {
    // Mapping from user address and content ID to stream expiry timestamp
    mapping(address => mapping(string => uint256)) public streamExpiry;

    /**
     * @dev Check if a user has active access to content
     * @param user Address of the user
     * @param contentId Unique identifier for the content
     * @return bool True if user has active access
     */
    function canStream(address user, string memory contentId) 
        public view returns (bool) 
    {
        return streamExpiry[user][contentId] > block.timestamp;
    }
    
    // Additional contract functions...
}
